# Othello-AI
黑白棋AI
*   **PART1：项目背景：**
    黑白棋是一款在 8×8 棋盘 上进行的双人策略游戏，目标是让己方颜色的棋子在棋盘上占据多数。
    黑白棋（Othello/Reversi）作为一种经典的零和博弈游戏，其棋盘状态空间复杂、动态博弈特征明显，是检验算法搜索效率与决策质量的理想平台。本项目旨在开发一套高性能的智能对战系统。通过设计并实现极小极大（Minimax）算法及 Alpha-Beta 剪枝策略，解决在有限计算资源下如何实现最优走法决策的问题，并深入探索启发式评估函数在复杂棋局状态下对 AI 决策水平的影响。
    目标：将课堂所学的搜索、剪枝、评估与规则约束落地为可运行、可测试、可演示的完整小项目。
黑白棋具体规则如下：
### 一、棋具与初始布局

| 项目 | 说明 |
|:-----|:-----|
| 棋盘 | 8×8方格，共64格 |
| 棋子 | 64枚双面棋子（一面黑、一面白），下在 **格子内** 而非交叉点上 |
| 初始布局 | 棋盘正中央4格交叉放置 |
| 先行方 | <mark>黑方先下</mark> |

### 二、合法落子规则
2.1 核心原则：必须能翻转
合法落子的唯一条件：落子的位置必须能在 横、竖、斜八个方向 中，至少翻转对手一个或多个棋子。

2.2 翻转判定流程
text
新落棋子 → 沿8个方向检查 → 必须满足：
1. 方向上有连续的一个或多个对手棋子
2. 这些对手棋子之后必须是己方棋子（形成 夹击 ）
3. 被 夹住 的位置全部是对手棋子，不能有空格

2.3 翻转的强制性
<mark>一步棋在多个方向上同时满足翻转条件时，所有被夹住的棋子都必须翻转</mark>，棋手无权选择不翻某些棋子。

### 三、弃权（Pass）规则

| 情况 | 处理方式 |
|:-----|:---------|
| 当前玩家 **没有** 合法落子位置 | <mark>必须弃权</mark>，由对手继续下 |
| 当前玩家 **有** 合法落子位置 | <mark>必须落子</mark>，不得弃权 |
| 双方都连续弃权 | 游戏结束 |


### 四、胜负判定

| 游戏结束条件 | 说明 |
|:------------|:-----|
| 棋盘填满 | 64格全部占满 |
| 双方都无合法走法 | 棋盘尚有空格，但无人能下 |

**胜负计算：** 游戏结束时，**棋子多的一方获胜**。

**特殊情况：** 如果一方的棋子被对方吃光（棋盘未满），棋局立即结束，<mark>吃光对方者获胜</mark>。
*   **PART2：核心算法：**
### 一、搜索策略：Alpha-Beta 剪枝
Alpha-Beta剪枝是在 Minimax 搜索基础上的优化算法，用于在保证找到最优解的前提下，剔除不需要搜索的 无用分支 。

实现逻辑： 在 alpha_beta() 函数中，通过 alpha（当前已知最大值，下界）和 beta（当前已知最小值，上界）维护一个搜索窗口。如果某个节点的估值超出了此窗口，则停止继续搜索该节点的子分支，<mark>大幅减少搜索空间</mark>。效率提升可达 100–1000 倍。

### 二、启发式评估函数

评估函数是AI决策过程中最为关键的部分，通过 `evaluate()` 对当前棋局进行多维度加权评分。本程序采用 **10 个维度** 的综合评估模型：

| 维度 | 权重 | 说明 |
|:----:|:----:|:-----|
| 棋子数量差 | 6~15（动态） | 基础指标，<mark>残局阶段权重加大</mark> |
| 角落控制 | ×2（30分/角） | <mark>黑白棋最重要的位置</mark>，永不翻转 |
| 稳定子 | 15 | 无法被翻转的棋子，决定终局优势 |
| 移动性 | 10 | AI走法越多越好，对手越少越好 |
| 边缘棋子 | -5 | 边缘棋子过多容易被对手利用 |
| 中心控制 | 1.5 | 中心棋子控制范围大 |
| 奇偶性 | 3（残局） | 控制最后一步的主动权 |
| 边控制 | 5 | 占据边路限制对手 |
| 潜在翻转数 | 0.5 | 当前走法未来的翻转潜力 |
| 空位分布 | 2 | 避免下在中心附近的坏位置 |

### 三、效率优化策略

为了在 1.5–3秒 内完成高质量搜索，程序采用了以下关键技术：

#### ① 走法排序

在进入Alpha-Beta搜索前，通过 `get_sorted_moves()` 对所有合法走法进行预排序：

| 优先级 | 走法类型 | 加分 |
|:------:|:--------:|:----:|
| 最高 | 占角 | +10000 |
| 高 | 占边 | +100 |
| 惩罚 | 坏位置（离角一格） | -200 |
| 中 | 占中心 | +30 |
| 低 | 翻转数量多 | +5/个 |

<mark>优先搜索高收益分支，使Alpha-Beta剪枝更早、更频繁地发生</mark>。

#### ② 动态深度策略
根据棋盘剩余空格数动态调整搜索深度：

| 空位数 | 搜索深度 | 阶段 |
|:------:|:--------:|:----:|
| > 50 | 6层 | 开局 |
| 40–50 | 7层 | 中局前期 |
| 30–40 | 8层 | 中局后期 |
| 15–30 | 9层 | 残局前期 |
| < 15 | 10层 | 残局精确搜索 |

#### ③ 超时控制
通过 `set_timeout()` 和 `time_exceeded()` 机制，确保AI在设定的时间限制内返回搜索结果，防止界面卡死：

| 阶段 | 时间限制 |
|:-----|:--------:|
| 中局 | 1.5秒 |
| 残局（空位 < 20） | 2秒 |
| 残局（空位 < 10） | 3秒 |

#### ④ 动态截断
当某层搜索分支过多（>15个）且深度较深时，<mark>只选择优先级最高的前15个走法</mark>进行搜索，避免搜索树爆炸。
### 四、棋局逻辑处理

| 功能 | 函数 | 说明 |
|------|------|------|
| 规则模拟 | `apply_move()` / `get_valid_moves()` | 严谨实现 必须翻转对手棋子 的规则 |
| 状态管理 | `make_move()` / `undo_move()` | 配合 `copy_board()` 实现棋盘深拷贝与还原 |
| 异常处理 | 弃权逻辑 | 一方无走法时自动跳过，<mark>双方都无走法时触发 game_over</mark> |

*   **PART3：运行指南：**

（本项目在GUI层面实现了棋子高光渲染、绿色落子提示、当前玩家指示器等视觉优化，并通过“悔棋”、“弃权”、“重新开始”等按钮完成交互功能。）

开始对局之后，玩家需要在界面上绿色落子提示中选择一步，与AI进行交互，直到游戏结束，若发现有误触或者想回溯的步骤，可点击右方的悔棋功能进行回溯

在游戏过程中，若轮到玩家但并未绿色落子提示，且还有空格，则玩家需点击弃权按钮（这也是黑白棋的规则之一）或者回溯重新选择，使得游戏继续下去
